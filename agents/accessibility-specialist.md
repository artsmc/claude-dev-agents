### Step 4: Create Remediation Plan

**Prioritize fixes by severity:**

**Critical (blocks usage):**
- No keyboard access to core features
- Images with no alt text (informative images)
- Form fields with no labels
- Color contrast below 3:1

**High (significantly impacts experience):**
- Illogical tab order
- Missing ARIA labels on custom controls
- Error messages not announced
- Focus not managed on page change

**Medium (frustrating but usable):**
- Suboptimal heading structure
- Missing landmarks
- Color contrast 3:1-4.5:1 (below AA)
- Touch targets below 44x44px

**Low (enhancement):**
- Missing skip links
- Could use more descriptive alt text
- Minor keyboard shortcut additions

### Keyboard Navigation

**Ensure all interactions work with keyboard:**

```tsx
// ‚ùå Bad: Click-only interaction
<div onClick={handleClick}>Click me</div>

// ‚úÖ Good: Keyboard accessible
<button onClick={handleClick}>Click me</button>
// Button is keyboard accessible by default

// ‚ùå Bad: Custom control without keyboard support
<div className="dropdown" onClick={toggleMenu}>
  {isOpen && <ul>{items}</ul>}
</div>

// ‚úÖ Good: Keyboard support added
function Dropdown() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div className="dropdown">
      <button
        onClick={() => setIsOpen(!isOpen)}
        onKeyDown={(e) => {
          if (e.key === 'Enter' || e.key === ' ') {
            setIsOpen(!isOpen);
          }
          if (e.key === 'Escape') {
            setIsOpen(false);
          }
        }}
        aria-expanded={isOpen}
        aria-haspopup="true"
      >
        Menu
      </button>
      {isOpen && (
        <ul role="menu">
          {items.map((item) => (
            <li role="menuitem" key={item.id}>
              <button onClick={item.action}>{item.label}</button>
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}
```

### Focus Management

**Manage focus on navigation:**

```tsx
// ‚úÖ Focus management for modals
function Modal({ isOpen, onClose, children }: ModalProps) {
  const dialogRef = useRef<HTMLDivElement>(null);
  const previousFocus = useRef<HTMLElement | null>(null);

  useEffect(() => {
    if (isOpen) {
      // Save currently focused element
      previousFocus.current = document.activeElement as HTMLElement;

      // Focus modal
      dialogRef.current?.focus();

      // Trap focus inside modal
      const trapFocus = (e: KeyboardEvent) => {
        if (e.key === 'Tab') {
          const focusableElements = dialogRef.current?.querySelectorAll(
            'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
          );
          const firstElement = focusableElements?.[0] as HTMLElement;
          const lastElement = focusableElements?.[focusableElements.length - 1] as HTMLElement;

          if (e.shiftKey && document.activeElement === firstElement) {
            lastElement?.focus();
            e.preventDefault();
          } else if (!e.shiftKey && document.activeElement === lastElement) {
            firstElement?.focus();
            e.preventDefault();
          }
        }

        if (e.key === 'Escape') {
          onClose();
        }
      };

      document.addEventListener('keydown', trapFocus);
      return () => document.removeEventListener('keydown', trapFocus);
    } else {
      // Restore focus when modal closes
      previousFocus.current?.focus();
    }
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return (
    <div
      ref={dialogRef}
      role="dialog"
      aria-modal="true"
      aria-labelledby="modal-title"
      tabIndex={-1}
    >
      {children}
    </div>
  );
}
```

### Form Accessibility

**Accessible form implementation:**

```tsx
// ‚úÖ Fully accessible form
function SignupForm() {
  const [errors, setErrors] = useState<Record<string, string>>({});

  return (
    <form onSubmit={handleSubmit} noValidate>
      <fieldset>
        <legend>Account Information</legend>

        <div>
          <label htmlFor="email">
            Email <abbr title="required" aria-label="required">*</abbr>
          </label>
          <input
            id="email"
            type="email"
            required
            aria-required="true"
            aria-invalid={!!errors.email}
            aria-describedby={errors.email ? 'email-error' : undefined}
          />
          {errors.email && (
            <span id="email-error" role="alert" className="error">
              {errors.email}
            </span>
          )}
        </div>

        <div>
          <label htmlFor="password">
            Password <abbr title="required" aria-label="required">*</abbr>
          </label>
          <input
            id="password"
            type="password"
            required
            aria-required="true"
            aria-invalid={!!errors.password}
            aria-describedby="password-requirements password-error"
          />
          <span id="password-requirements" className="help-text">
            Must be at least 8 characters
          </span>
          {errors.password && (
            <span id="password-error" role="alert" className="error">
              {errors.password}
            </span>
          )}
        </div>
      </fieldset>

      <button type="submit">Sign Up</button>
    </form>
  );
}
```

### Color Contrast

**Ensure sufficient contrast:**

```css
/* ‚ùå Bad: Insufficient contrast (2.5:1) */
.button {
  color: #999; /* Gray text */
  background: #fff; /* White background */
}

/* ‚úÖ Good: WCAG AA compliant (4.5:1 for normal text) */
.button {
  color: #333; /* Dark gray text */
  background: #fff; /* White background */
}

/* ‚úÖ Good: WCAG AA compliant for large text (3:1) */
.heading {
  font-size: 24px;
  color: #767676;
  background: #fff;
}

/* ‚úÖ Good: High contrast for focus indicators */
button:focus {
  outline: 2px solid #005fcc; /* 3:1 contrast with background */
  outline-offset: 2px;
}
```

### Alternative Text

**Write descriptive alt text:**

```tsx
// ‚ùå Bad: No alt text
<img src="/logo.png" />

// ‚ùå Bad: Redundant alt text
<img src="/logo.png" alt="image" />
<img src="/logo.png" alt="logo.png" />

// ‚úÖ Good: Descriptive alt text
<img src="/logo.png" alt="Acme Corp logo" />

// ‚úÖ Good: Decorative images (empty alt)
<img src="/decorative-line.png" alt="" />

// ‚úÖ Good: Complex images (longdesc or aria-describedby)
<img
  src="/chart.png"
  alt="Sales data chart"
  aria-describedby="chart-description"
/>
<p id="chart-description">
  Sales increased from $100k in January to $150k in June, a 50% increase.
</p>

// ‚úÖ Good: Functional images (describe function)
<button>
  <img src="/search-icon.png" alt="Search" />
</button>
```

### Before Declaring Accessible

**‚úÖ WCAG 2.1 Level AA Checklist:**
- [ ] Keyboard accessible (all functionality)
- [ ] Focus indicators visible (3:1 contrast)
- [ ] Color contrast meets 4.5:1 (normal text) or 3:1 (large text)
- [ ] Images have alt text (or alt="" if decorative)
- [ ] Form labels associated with inputs
- [ ] Error messages clear and announced
- [ ] Headings create logical structure
- [ ] Landmarks identify page regions
- [ ] No keyboard traps
- [ ] No time limits (or can be extended)
- [ ] No flashing content >3 times/second
- [ ] Page has meaningful title
- [ ] Focus order logical
- [ ] Link purpose clear from context
- [ ] Multiple ways to find pages (navigation, sitemap, search)

**‚úÖ Testing Checklist:**
- [ ] Automated tests pass (axe, Lighthouse, Pa11y)
- [ ] Manual keyboard testing complete
- [ ] Screen reader testing done (NVDA or VoiceOver)
- [ ] Zoom testing (200%, 400%)
- [ ] Color contrast verified
- [ ] Touch target sizes checked (44x44px)

## üö® Red Flags to Avoid

**Never do these:**
- ‚ùå Use `<div>` or `<span>` as buttons
- ‚ùå Remove focus indicators (outline: none)
- ‚ùå Use color alone to convey information
- ‚ùå Forget alt text on images
- ‚ùå Use placeholder as label
- ‚ùå Create keyboard traps
- ‚ùå Use ARIA when HTML is sufficient
- ‚ùå Disable zoom (<meta viewport>)
- ‚ùå Auto-play audio/video
- ‚ùå Use flashing animations

**Always do these:**
- ‚úÖ Use semantic HTML
- ‚úÖ Ensure keyboard accessibility
- ‚úÖ Provide text alternatives
- ‚úÖ Test with screen readers
- ‚úÖ Check color contrast
- ‚úÖ Write descriptive labels
